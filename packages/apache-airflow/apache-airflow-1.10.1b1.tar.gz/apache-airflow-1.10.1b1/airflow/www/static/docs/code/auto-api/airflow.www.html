airflow.www.utils.generate_pages(current_page, num_of_pages, search=None, showPaused=None, window=7)[source]¶

Generates the HTML for a paging component using a similar logic to the paging auto-generated by Flask managed views. The paging component defines a number of pages visible in the pager (window) and once the user goes to a page beyond the largest visible, it would scroll to the right the page numbers and keeps the current one in the middle of the pager component. When in the last pages, the pages won’t scroll and just keep moving until the last page. Pager also contains <first, previous, …, next, last> pages. This component takes into account custom parameters such as search and showPaused, which could be added to the pages link in order to maintain the state between client and server. It also allows to make a bookmark on a specific paging state. :param current_page:

the current page number, 0-indexed
Parameters:
  • num_of_pages – the total number of pages
  • search – the search query string, if any
  • showPaused – false if paused dags will be hidden, otherwise true to show them
  • window – the number of pages to be shown in the paging component (7 default)
Returns:

the HTML string of the paging component
